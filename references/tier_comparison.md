## 适用场景说明

### 🟢 Basic 版（基础版）

**适合场景：**
- 日常数据查询和简单分析
- 短期趋势预测（7-30 天）
- 快速验证想法或假设
- 资源受限环境（如个人笔记本）
- 学习和探索性数据分析

**典型用例：**
```
/duckdb 分析 上个月销售额
/duckdb 运行预测 未来 7 天的日活用户
/duckdb 记录 查看昨天的查询
```

**不建议用于：**
- 需要高精度的业务决策
- 复杂的多变量预测
- 需要模型解释的场景

---

### 🔵 Plus 版（增强版）

**适合场景：**
- 生产环境的数据分析
- 需要模型可解释性的业务场景
- 中短期预测（30-90 天）
- 需要自动特征选择的场景
- 异常检测和离群点分析

**典型用例：**
```
/duckdb 运行预测 下季度营收 (自动特征工程)
/duckdb 分析 用户流失原因 (SHAP 解释)
/duckdb 运行预测 库存需求 (含异常检测)
```

**核心优势：**
- **自动特征工程**：自动识别和构造有效特征
- **交叉验证**：更可靠的模型评估
- **SHAP 解释**：理解模型决策依据
- **异常检测**：识别数据中的异常模式

---

### 🟣 Pro 版（专业版）

**适合场景：**
- 复杂时间序列预测（>90 天）
- 高精度要求的业务预测
- 多变量、非线性关系建模
- 因果推断和策略评估
- 持续学习和模型自适应

**典型用例：**
```
/duckdb 运行预测 年度销售趋势 (深度学习模型)
/duckdb 分析 营销活动效果 (因果推断)
/duckdb 运行预测 股价走势 (LSTM/Transformer)
```

**核心优势：**
- **深度学习模型**：LSTM、Transformer 等先进架构
- **因果推断**：评估干预措施的真实效果
- **持续进化**：模型随新数据自动更新
- **最高精度**：适合关键业务决策

## 升级路径

### 自动升级

系统支持**自动升级**机制（`auto_upgrade: true`）：

1. **Basic → Plus**：当检测到以下情况时自动建议升级
   - 用户请求特征重要性分析
   - 需要模型解释
   - 数据包含异常值需要检测

2. **Plus → Pro**：当检测到以下情况时自动建议升级
   - 用户请求深度学习模型
   - 需要因果推断分析
   - 预测周期超过 90 天

### 手动升级

用户可随时通过命令手动切换版本：

```bash
# 查看当前版本
/duckdb 配置 查看版本

# 升级到 Plus 版
/duckdb 配置 升级 plus

# 升级到 Pro 版
/duckdb 配置 升级 pro

# 降级到 Basic 版
/duckdb 配置 降级 basic
```

### 版本切换流程

```
[用户请求升级]
    │
    ▼
[检查目标版本依赖] ── 缺失 ──→ [自动安装依赖]
    │
   已安装
    │
    ▼
[确认升级] ──→ [更新配置] ──→ [重启服务]
    │
    ▼
[升级完成，版本切换]
```

## 常见问题

### Q: 我可以随时切换版本吗？
A: 是的，可以随时升级或降级。降级不会影响已有数据，但可能无法使用高级功能。

### Q: 升级需要额外费用吗？
A: 技能本身免费，但 Pro 版需要更多计算资源，可能增加云资源成本。

### Q: 如何查看当前版本？
A: 运行 `/duckdb 配置 查看版本` 即可显示当前版本信息。

### Q: 自动升级会中断我的任务吗？
A: 不会，自动升级只在任务间隙进行，或在用户确认后执行。
